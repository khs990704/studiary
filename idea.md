## 1. 문서 목적

본 문서는 학습 기록 및 세션 관리 중심의 공부 기록 서비스 화면 구조와 기능 범위를 정리한 것이다.  
기획 설계 에이전트가 실제 화면 설계, 기능 명세, 시스템 구조, 배포 구조까지 확장할 수 있도록  
와이어프레임 기준의 핵심 흐름과 시스템 역할을 문서화하는 것을 목적으로 한다.

따라서 본 문서는 다음을 준수한다.

- 화면 간 이동 흐름이 명확할 것
- 각 화면의 핵심 기능이 분명할 것
- 사용자 입력과 시스템 생성 결과의 연결 구조가 드러날 것
- 세션 시작, 진행, 수정, 종료 이후 리뷰까지의 흐름이 일관될 것
- AI 요약 및 피드백 기능이 MVP 범위에 포함될 것
- 실제 배포를 고려한 구조가 반영될 것

---

## 2. 서비스 방향

본 서비스는 단순히 공부 시간을 측정하는 타이머 앱이 아니라,  
각 공부 세션에서 사용자의 집중도와 방해요소를 기록하고, 이를 기반으로 AI 요약과 피드백까지 제공하는  
**학습 세션 생산성 기록 및 요약/피드백 서비스**를 제공한다.

본 서비스는 다음 흐름을 하나의 제품 경험으로 연결하는 데 초점을 둔다.

- 공부 시작 전 진입
- 공부 중 세션 기록 
- 공부 종료 후 학습 진행이 얼마나 효율적으로 잘 되었는지 기록
- AI 기반 요약 및 피드백 확인

즉, 본 서비스의 핵심은  
공부 시간 기록 자체에 그치지 않고  
기록된 공부 진행 과정중 기록된 집중도/방해요소 데이터를 바탕으로 사용자가 자신의 학습 상태를 복기하고 개선할 수 있도록 돕는 것이다.

---

## 3. 핵심 구조

서비스는 크게 여섯 화면 흐름으로 구성된다.

1. 기록된 공부 목록 메인 페이지
2. 공부 기록 페이지
	1. 오늘 공부 시작 전 초기 
	2. 세션 추가 
	3. 공부 진행 중 및 세션 수정 
	4. 공부 종료 및 리뷰 

또한 시스템은 아래 두 기능 축이 함께 동작하는 구조를 가진다.

1. 사용자 공부 세션 기록 및 조회
2. OpenRouter 기반 AI 요약 및 피드백 생성

이 두 기능은 분리된 부가 요소가 아니라  
초기 MVP 단계부터 함께 동작해야 하는 핵심 구조로 본다.

## 4. 핵심 기능

### 4.1. 기록된 공부 목록 (메인 페이지)

사용자가 이전에 기록한 공부 내용을 한눈에 확인하는 메인 화면이다.  
상단에는 날짜 기반(달별) 히트맵을 두고, 하단에는 날짜별(일별) 공부 기록 카드 목록을 제공한다.

#### wireframe
```text
+--------------------------------------------------+
|  Recorded Studies                                |
+--------------------------------------------------+  
|  [ Study Heatmap ]                               |
|  |2024|04|                                       | <--- 년도,월 선택 가능(기본값 현재 날짜)
|  [ ][ ][ ][x][x][ ][ ][ ][ ][ ][ ][ ][ ][ ]      | <--- 해당 월 히트맵 표시
|  [ ][x][x][x][ ][ ][ ][ ][ ][ ][ ][ ][ ][ ]      |
|  (잔디 색상: 집중도에 따라 옅은/진한 초록색)               |
|                                                  |
|   2026-04-09                                     |
|  +--------------------------------------------+  |
|  | 총 공부시간: 4h 30m (집중: 4h, 휴식: 30m)       |  | <--- Recorded Study Card 
|  | AI 요약: 스트레칭 후 공부 시작. 컨디션 굿           |  |
|  | AI 피드백: 스트레칭 이후 방 환기                  |  |
|  +--------------------------------------------+  | 
|                                                  |
|   2026-04-10                                     |
|  +--------------------------------------------+  |
|  | 총 공부시간: 3h 15m (집중: 3h, 휴식: 15m)       |  |
|  | AI 요약: 생각이 많음. 배가 고픔.                 |  |
|  | AI 피드백: 생각을 노트에 정리. 간식 준비.          |  |
|  +--------------------------------------------+  |
|                                                  |
|   2026-04-11                                     | <--- 현재 날짜 
|  +--------------------------------------------+  |
|  |                                            |  | <--- 현재 날짜 아래에는 공부하러가기 버튼
|  |               [ 공부 하러 가기 ]              |  |
|  |                                            |  |
|  +--------------------------------------------+  |
+--------------------------------------------------+
```

#### 세부 기능

**Study Heatmap**
- 항상 상단 고정
- 선택된 년/월(기본값은 현재 날짜) 의 일일 집중도를 히트맵으로 표시
	- 일별 스터디 세션 **집중도의 평균의 올림 값**에 따라 색상 강도 표시
	- 집중도 1~5단계에 따라 옅은 **초록 ~ 진한 초록**으로 표현
	- 해당 날짜에 공부 기록이 없거나 집중도 평균이 0인 경우 갈색으로 표현
- heatmap에서 특정 날짜 클릭 시 선택된 날짜의 Recorded Study Card가 스크롤 영역의 상단으로 오도록 부드러운 스크롤 모션

**Recorded Study Card**
- 스크롤 가능 영역
- 날짜별 공부 기록 요약 표시되는 영역(heatmap에서 선택된 년/월의 1일부터 말일까지 표시하되, 기록이 없는 날은 제외한다)
	- 공부 기록이 된 날짜의 경우
		- 총 공부 시간(총 집중 시간 + 총 휴식 시간), 총 집중 시간(공부 시간의 합), 총 휴식 시간(휴식 시간의 합) 표시
		- AI 요약, AI 피드백 표시
	- 현재 날짜의 경우 
		- 공부하러가기 버튼 표시
- 카드 클릭 시 해당 날짜의 공부 기록 화면으로 이동

#### 기능 의도

이 화면은 단순 목록 열람이 아니라,  
사용자가 자신의 학습 흐름을 빠르게 확인하고  
날짜별 기록과 AI 리뷰 결과를 함께 확인하는 기록 허브 역할을 가진다.

### 4.2 공부 기록 페이지
### 4.2.1 오늘 공부 시작 전 초기 페이지

하루 공부를 시작하기 전 진입 화면이다.  
아직 세션이 없는 상태에서 오늘 공부를 시작할 수 있도록 진입점을 제공한다.

#### wireframe
```text
+--------------------------------------------------+
|  [<] YYYY-MM-DD                                  | <--- Date Header
+--------------------------------------------------+
|                                                  |
|                       (+)                        | <--- Add Session Timer
|                                                  |
|                                                  |
|                                                  |
|                                                  |
|                                                  |
|                                                  |
|                                                  |
|                                                  |
|                                                  |
|                                                  |
|                 [ 오늘 공부 끝내기 ]                 | <--- Bottom Action
+--------------------------------------------------+
```

#### 세부 기능

**Date Header** (`< YYYY-MM-DD`)
- 오늘 날짜 표시
- 메인 화면으로 이동 가능한 버튼

**Add Session Timer** (`+` (동그란 모양의 더하기 버튼))
- 클릭시 Timer Setup 표시

**Bottom Action**
- “오늘 공부 끝내기” 버튼
- 당일 공부 세션이 하나도 없다면 세션이 없다는 알림만 반환
- 당일 공부 세션이 있는 경우 공부 리뷰 및 피드백 생성

#### 기능 의도

이 화면은 사용자가 오늘의 공부를 시작하기 위한 가장 단순한 진입 페이지이며,  
아직 기록이 없는 상태를 자연스럽게 처리하는 시작점 역할을 한다.

---

### 4.2.2 세션 추가 페이지

새로운 공부 세션을 생성하는 화면이다.  
공부 시작 전 타이머를 설정하고 세션을 시작하는 기능을 담당한다.

#### wireframe
```text
+--------------------------------------------------+
|  [<] YYYY-MM-DD                                  |
+--------------------------------------------------+
|                                                  |
|  +--------------------------------------------+  |
|  |                   [ ▲ ]                    |  |
|  |                   50:00                    |  | <--- Timer Setup
|  |                   [ ▼ ]                    |  |
|  |                                            |  |                       
|  |                  [ 시작 ]                   |  | <--- Session Start
|  +--------------------------------------------+  |
|                                                  |
|                                                  |
|                                                  |
|                                                  |
|                                                  |
|                 [ 오늘 공부 끝내기 ]                 |
+--------------------------------------------------+
```

#### 세부 기능

**Date Header** (`< YYYY-MM-DD`)
- 오늘 날짜 표시
- 메인 화면으로 이동 가능한 버튼

**Timer Setup**
- 타이머 시간 조절 UI 제공 (직접 입력, 스크롤 조절, 업다운 버튼 클릭으로 1min씩 조절)

**Session Start**
- 타이머 시작 후 session이 생성됨 (위 wireframe 상에서는 직전 세션이 없으므로 공부 session이 생성됨)
	- 직전 세션이 없다면 공부 session 생성
	- 직전 세션이 공부라면 휴식 session 생성
	- 직전 세션이 휴식이라면 공부 session 생성

**Bottom Action**
- “오늘 공부 끝내기” 버튼
- 당일 공부 세션이 하나도 없다면 세션이 없다는 알림만 반환
- 당일 공부 세션이 있는 경우 공부 리뷰 및 피드백 생성

#### 입력 요소

- 공부session(또는 휴식session) 시간 설정
- session 타이머 시작

#### 기능 의도

이 화면은 공부를 실제 기록 가능한 단위인 session으로 전환하는 시작 단계이며,  
사용자가 별도 복잡한 입력 없이 바로 공부를 시작할 수 있도록 단순하게 설계한다.

### 4.2.3 공부 진행 중 및 세션 수정 페이지

세션이 진행 중일 때 상태를 기록하는 화면이다.  
이미 진행 중이거나 생성된 세션 정보를 수정할수도 있는 화면이다.  
세션의 상태와 기록값을 빠르게 조정할 수 있도록 한다.
현재 세션의 시간 흐름과 집중 상태, 방해 요소를 함께 기록한다.

#### wireframe
```text
+--------------------------------------------------+
|  [<] YYYY-MM-DD (오늘 날짜)                        |
+--------------------------------------------------+
|                                                  |
|  [ 세션 1 ]                                       |
|  +---------+                                [⋮]  | <- 클릭 시 삭제 메뉴 표시
|  |         |  집중도: (•) (•) (•) ( ) ( )          | <- Focus Tracking
|  |  45:59  |  방해요소:                            | <- Session Running State
|  | ( II )  |  +---------------------------------+| <- Distraction Logging
|  |         |  | 친구랑 싸워서 신경쓰여서 집중이 안됨.    ||
|  +---------+  +-------------------------(34/100)+|
|                                                  |
|                      (+)                         | 
|                                                  |
|                                                  |
|                                                  |
|                                                  | 
|                                                  |
|                 [ 오늘 공부 끝내기 ]                 |
+--------------------------------------------------+
```

#### 세부 기능

**Date Header** (`< YYYY-MM-DD`)
- 오늘 날짜 표시
- 메인 화면으로 이동 가능한 버튼

**Session Running State**
- 세션 유형을 시각적으로 구분하는 원형(휴식 시간) 또는 사각형(공부 시간) UI 표시
	- 진행 중인 타이머 시간 표시
	- 타이머 재생/일시정지 기능 
		- 해당 재생 또는 일시정지는 사각형 또는 원형 UI 백그라운드(타이머 시간의 뒷배경)에 현재 상태의 반대되는 상태 아이콘(타이머 재생 중에는 일시정지, 일시정지 중에는 재생)이 희미하게 표시되다가 마우스 커서를 올릴 경우에만 진하게 표시됨 
- 세션UI는 항상 사각형 (공부 시간) 다음 원형 (휴식 시간) 순으로 구성
- 시간이 다 채워진 경우에는 사각형 (학습) 의 경우 study 문구와 그 밑에 완료한 학습 시간, 원형 (휴식)의 경우 rest 문구와 그 밑에 완료한 휴식 시간으로 변경됨
- 세션의 시간이 다 채워져 변경된 후에는 무조건 새로운 세션을 추가하거나 "오늘 공부 끝내기" 버튼 클릭, 재시작은 허용하지 않음

**Session Edit**
- 모든 session 의 Session Edit 버튼은 당일까지만 활성화
- 해당 세션 우측의 점 세개(`⋮`) 버튼 클릭 시 "세션 삭제" 툴팁 메뉴를 점 세개(`⋮`) 버튼 좌측에 표시. "세션 삭제" 클릭 시 세션을 삭제할것인지 묻는 알림창 표시
- 삭제 여부 알림창에서 "확인" 클릭 시 해당 세션 삭제
- 삭제 여부 거절(알림창 X 버튼, esc 키, 여백 클릭) 시 세션 삭제 여부를 묻는 알림창 제거

**Focus Tracking**
- 공부 Session Running State UI인 사각형 컴포넌트의 오른쪽 영역 좌상단에 위치 (휴식 session에는 미표시)
- 집중도 선택
	- 1~5단계 수준으로 기록 가능 (특정 단계 선택시 이전 단계들도 다 채색되는 프로그레스 바 형태)
	- 초기 기록 후 "오늘 공부 끝내기" 버튼 클릭 전까지 수정 가능

**Distraction Logging**
- Focus Tracking 하단에 위치 (휴식 session에는 미표시)
- 방해 요소 입력
	- 내부적/외부적 방해 요소를 간단히 텍스트로 기록
	- 입력 길이는 짧은 메모 수준으로 제한 (최대 100자)
	- 초기 기록 후 "오늘 공부 끝내기" 버튼 클릭 전까지 수정 가능

**Add Session Timer** (`+` (동그란 모양의 더하기 버튼))
- 클릭시 Timer Setup 표시

**Bottom Action**
- “오늘 공부 끝내기” 버튼
- 당일 공부 세션이 하나도 없다면 세션이 없다는 알림만 반환
- 당일 공부 세션이 있는 경우 피드백 생성 
	- 피드백 생성 성공시 모든 session 의 Session Edit 버튼을 비활성화

#### 기능 의도

이 화면은 단순 타이머 화면이 아니라,  
공부 도중의 집중 상태와 방해 요소를 기록하여  
하루 종료 시 AI가 요약과 피드백을 생성할 수 있는 기초 데이터로 축적하는 역할을 한다.

### 4.2.4 공부 종료 및 리뷰 페이지

하루 공부를 종료한 뒤 전체 세션 목록과 리뷰를 확인하는 화면이다.  
오늘의 공부 흐름을 정리하고, AI 요약과 피드백까지 함께 제공하는 역할을 한다.

#### wireframe
```text
+--------------------------------------------------+
|  [<] YYYY-MM-DD (오늘 공부 리뷰)                    |
+--------------------------------------------------+
|                                                  |
|  [ 세션 1 ]                                       | <- Session List
|  +---------+                                [⋮]  |
|  |         |  집중도: (•) (•) (•) ( ) ( )          |
|  |  study  |  방해요소:                            | 
|  |  50min  |  +---------------------------------+|
```

#### 세부 기능

**Session List**
- 오늘 생성된 study / rest 세션 목록을 순서대로 표시

**Daily Review Chart**
- Session List 스크롤을 다 내렸을때 그 마지막 하단에 위치 
- 오늘의 집중도 변화 그래프 제공
- 세션 순서에 따른 집중도 변화 시각화
- x축은 세션 번호, y축은 각 세션의 집중도

**AI Review**
- 오늘 공부에 대한 AI 요약(집중도, 방해요소, 학습 및 휴식 시간 등 당일 학습의 전반적인 내용) 제공

**AI Feedback**
- AI Review를 기반으로 다음 공부를 위한 피드백(방해 요소 개선 또는 학습 패턴 관련 제안) 제공

이 화면은 하루 공부의 종료 지점이자 복기 화면이며,  
기록된 세션 데이터를 바탕으로  
AI가 요약과 피드백을 생성하여  
사용자가 오늘의 학습 패턴을 확인하고 다음 공부에 반영하도록 돕는 핵심 화면이다.

## 5. AI 활용 계획

### 5.1. MVP 포함 범위

AI 기능은 초기 MVP에 반드시 포함된다.  
즉, MVP는 단순 세션 기록 기능만 동작하는 상태가 아니라  
**세션 기록 + AI 요약/피드백 생성까지 완료된 상태**를 기준으로 한다.

### 5.2. 사용 방식

AI 기능은 외부 모델 서버를 직접 운영하지 않고  
**OpenRouter API**를 통해 호출한다.

초기에는 **무료 모델**을 사용하여  
하루 공부 세션 기록에 대한 요약 및 피드백을 생성한다.
생성하는 결과가 두 가지 이므로 요청은 각각 총 2번을 수행한다. (하루 공부 요약, 요약 기반 피드백)
"오늘 공부 끝내기" 버튼 클릭 시 두 api는 자동으로 순차적으로 호출됨
하루 공부 요약의 결과가 반환되지 않은 경우 요약 기반 피드백 api는 호출하지 않음
필요하고 빠른 응답만 받기 위해, 모델의 `reasoning`은 False인 상태로 진행

### 5.3. AI 출력 범위

AI는 다음 범위의 결과를 생성한다.
1. 하루 공부 요약 (집중도, 방해요소, 학습 및 휴식 시간 등 당일 학습의 전반적인 내용)
2. 공부 요약 기반 피드백 (방해 요소 개선 또는 학습 패턴 관련 제안)

### 5.4. AI 입력 데이터

AI는 범위 별 다음 사용자 데이터를 활용한다.
1. 하루 공부 요약
	- 모든 세션의 총 공부 시간
	- 모든 세션의 총 휴식 시간
	- 모든 세션의 평균 집중도의 올림값 (1 ~ 5)
	- 모든 세션의 각 방해 요소 텍스트 값들
2. 공부 요약 기반 피드백
	- 하루 공부 요약 텍스트

### 5.5. 적용 원칙

- AI는 자유 대화형 기능이 아니라 리뷰 생성 기능으로 사용한다.
- 출력은 짧고 읽기 쉬운 형태를 우선한다.
- 모델 교체 가능성을 고려하여 호출 구조는 분리한다.
- 무료 모델을 사용하되, 이후 필요 시 상위 모델로 교체 가능하도록 설계한다.
- 무료 모델의 경우 OpenRouter의 특성 상 응답을 못받을 것을 고려하여 메인 모델과 메인 모델이 실패할 경우 그 작업을 대신 처리한 fallback 용 모델까지 총 3개를 구성
	- `meta-llama/llama-3.1-8b-instruct:free`
	- `google/gemma-3-4b-it:free`
	- `google/gemma-3-1b-it:free`
- 혹여 openrouter가 잘 동작하지 않을 것을 대비하여, docker로 올라간 llama에 연동하는 것으로 대체할 것까지 고려하여 개발

## 6. 배포 고려 사항

본 서비스는 MVP 단계부터 실제 배포를 전제로 설계한다.

### 6.1. 배포 목표

초기 배포 버전에서는 아래 흐름이 실제 웹 환경에서 동작해야 한다.

- 메인 화면 진입
- 오늘 공부/휴식 세션 생성
- 공부 완료 후 기록 저장
- 세션 수정 및 학습 종료
- 학습 종료 후 AI 요약/피드백 생성 및 표시

### 6.2. 시스템 관점 고려 요소

- 프론트엔드와 백엔드가 분리 가능한 구조일 것
- 세션 기록 저장 및 지난 학습에서의 세션 및 요약/패드백 기록 저장을 위한 DB 연동 구조가 있을 것
- OpenRouter API 키는 서버 환경변수로 안전하게 관리할 것
- 배포 환경과 로컬 환경의 설정을 분리할 것
- 학습 종료 시 AI 호출 실패 시에도 기록 저장 자체는 유지되도록 설계할 것
- 학습 종료 시 AI 호출 실패 시 기록만 저장된 경우 나중에라도 AI 요약 및 피드백을 요청할 수 있도록 구성 (학습 종료 후 정보가 기록되었지만 AI 요약 및 피드백이 비어있음을 DB에 저장이 되어 추후 이를 기반으로 재 생성 버튼이 활성화 될 수 있도록 함)
- AI 기반 요약 및 피드백 재생성 버튼은 메인 페이지에서 기록은 있으나 AI 요약 및 피드백이 비어있는 경우에 한해 해당 기록 카드 클릭 후 로드된 공부 기록 페이지에 버튼이 활성화되도록 함 (생성이 된 경우에는 비활성화)

## 8. 설계 에이전트 전달 시 핵심 해석 포인트

### 8.1. 이 서비스의 본질

이 서비스는 단순 타이머 앱이 아니라  
**하루 공부 세션을 기록하고, AI 통해 요약 및 피드백을 받는 학습 세션 생산성 기록 및 요약/피드백 서비스**이다.

### 8.2. AI의 역할

AI는 후순위 부가 기능이 아니라  
초기 MVP부터 포함되는 핵심 기능이다.  
사용자가 남긴 세션 기록을 바탕으로  
요약과 피드백을 생성하는 역할을 수행한다.

### 8.3. 초기 제품 우선순위

초기 MVP에서는 아래 기능이 반드시 살아 있어야 한다.

- 날짜별 기록 확인
- 공부 세션 생성
- 세션 진행 상태 기록
- 집중도 입력
- 방해 요소 입력
- 세션 수정 및 삭제
- 하루 종료 후 리뷰 표시
- OpenRouter 기반 AI 요약/피드백 생성
- 실제 배포 가능한 최소 구조 확보
