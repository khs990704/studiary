# Idea Inquiry — 학습 세션 생산성 기록 및 AI 요약/피드백 서비스 (Studiary)

> 최종 업데이트: 2026-04-11  
> 상태: 🔄 구체화 중

---
## 아이디어 요약

하루 공부 세션을 기록하고, 집중도/방해요소 데이터를 바탕으로 AI가 요약과 피드백을 생성하는 학습 세션 생산성 기록 서비스.

## 해결하려는 문제

공부 시간 기록 자체에 그치지 않고, 기록된 집중도/방해요소 데이터를 바탕으로 사용자가 자신의 학습 상태를 복기하고 개선할 수 있도록 돕는다.

## 타겟 사용자

공부를 기록하고 개선하고 싶은 사용자 (구체적 세그먼트 확인 필요)

| 9 | 과거 기록 조회 | 메인에서 과거 날짜 카드 클릭 시 해당 날짜 기록 로드 | P0 |

## Claude의 질문

> 다음 질문에 답해주시면 설계 단계로 넘어갈 수 있습니다.

**Q1. 인증/사용자 관리가 필요한가요?**  
로그인 없이 개인 기기에서만 사용하는 구조인지, 아니면 계정 기반으로 여러 기기에서 접속 가능한 구조인지 알려주세요. MVP 단계의 범위만 결정해주시면 됩니다.  
→ 계정 기반으로 여러 기기에서 접속 가능한 구조 

**Q2. 선호하는 기술 스택이 있나요?**  
프론트엔드(React, Next.js 등), 백엔드(FastAPI, Spring Boot, Express 등), DB(PostgreSQL, SQLite, MongoDB 등) 중 선호하거나 이미 결정한 스택이 있으면 알려주세요. 없으면 idea.md 내용 기반으로 추천드리겠습니다.
→ 프론트엔드는 React, 상태관리에는 zustand, 백엔드는 FastAPI, DB는 PostgresSQL, DB migration은 Alembic

**Q3. 배포 환경은 어떻게 계획하고 계신가요?**  
idea.md에서 "실제 배포를 전제"로 언급하셨는데, 클라우드(AWS, GCP, Vercel 등), 자체 서버(Docker), 또는 기타 환경 중 어떤 것을 생각하고 계신가요?  
→ front - 3000 / back - 8200 / db - 5433 포트 번호를 지정해서 OCI 환경에서 docker로 배포할 거야. compose file 하나로 전부 배포될 수 있도록 해주고, 같은 네트워크로 묶어줘.
추가로 Cloudflare를 이용해 HTTPS를 적용할 예정이니까, Nginx는 외부(80번 포트)에서 HTTP 요청만 받아서 프론트와 백엔드로 라우팅(리버스 프록시)하도록 설정해 줘. SSL 인증서 세팅은 Nginx에서 안 해도 돼.
